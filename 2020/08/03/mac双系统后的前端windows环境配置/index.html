mac双系统后的前端windows环境配置
Created 2020-08-03| Updated 2020-08-03
Post View:

npm 全局变量的无法使用

  • 配置环境变量
    • 我的电脑属性>高级系统设置>环境变量>选择Path>修改路径为包含你的全局安装文件的npm文件夹（绝对路径）
  • 同意脚本的运行
    • 管理员打开window Powershell
    • 运行set-executionpolicy remotesigned
    • 后面选择 Y 或者 A

由于新的系统，原有的项目来无法启动。

  • 可以将node_module文件夹删除重新下载依赖包，解决此问题

vscode配置（这里是个人习惯）

  • 插件

    • Code Spell Checker
    • Vuetur
    • Vue 2es Snippets
    • Path Intellisense
    • JavaScript(ES6) code snippets
    • ESLint

  • 设置

    • 自动格式化开启（步骤 设置》正在格式化》format on save 勾选）

    • 配置字符串中可以自动补全(搜索settings.josn)

    • "emmet.triggerExpansionOnTab": true,
    "emmet.showAbbreviationSuggestions": true,
    "emmet.showExpandedAbbreviation": "always",
    "emmet.includeLanguages": {
        "javascript": "html"
    }

  • vue脚手架

  • express生成器

  • 其他按需下载这里就不一一列举了
