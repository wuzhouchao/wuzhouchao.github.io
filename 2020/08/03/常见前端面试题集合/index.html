  1. 谈谈你对作用域的理解？

    1. 作用域划分为 全局作用域 函数作用域 块级作用域
    2. 作用域本质就是变量的活动范围

  2. 什么是作用域链？

    1. 作用域链是由一个个嵌套作用域形成的一个访问序列的链条
    2. 就比如说一个变量首先进行当前作用域查找，当前没找到则向其上一层作用域进行查找，直到找到全局，如果还没有找到则报错，其寻找的路径就是作用域链

  3. 什么是闭包？

    1. 函数和对其周围状态（lexical environment，词法环境）的引用捆绑在一起构成闭包（closure）。也就是说，闭包可以让你从内部函数访问外部函数作用域。在 JavaScript 中，每当函数被创建，就会在函数生成时生成闭包。

    2. 就我个人的理解,函数内部数据，由于外部对其的引用，导致该数据在应该销毁的时候未被销毁，那么该函数就是闭包函数

    3. 广义 每个函数及其周围环境就是一个闭包

    4. 例如

      1. function makeAdder(x) {
  return function(y) {
    return x + y;
  };
}

var add5 = makeAdder(5);
var add10 = makeAdder(10);

console.log(add5(2));  // 7
console.log(add10(2)); // 12

      2. 在这个示例中，我们定义了 makeAdder(x) 函数，它接受一个参数 x ，并返回一个新的函数。返回的函数接受一个参数 y，并返回x+y的值。

        从本质上讲，makeAdder 是一个函数工厂 — 他创建了将指定的值和它的参数相加求和的函数。在上面的示例中，我们使用函数工厂创建了两个新函数 — 一个将其参数和 5 求和，另一个和 10 求和。

        add5 和 add10 都是闭包。它们共享相同的函数定义，但是保存了不同的词法环境。在 add5 的环境中，x 为 5。而在 add10 中，x 则为 10。

  5. vue中的v-if和v-show的区别？

    1. 实现方式
      1. v-if是根据后面数据的真假值判断直接从Dom树上删除或重建元素节点
      2. v-show只是在修改元素的css样式，也就是display的属性值，元素始终在Dom树上。
    2. 编译过程
      1. v-if切换有一个局部编译/卸载的过程，切换过程中合适地销毁和重建内部的事件监听和子组件；
      2. v-show只是简单的基于css切换；
    3. 编译条件
      1. v-if是惰性的，如果初始条件为假，则什么也不做；只有在条件第一次变为真时才开始局部编译；
      2. v-show是在任何条件下（首次条件是否为真）都被编译，然后被缓存，而且DOM元素始终被保留；
    4. 性能消耗
      1. v-if有更高的切换消耗，不适合做频繁的切换；
      2. v-show有更高的初始渲染消耗，适合做频繁的额切换；

  6. 什么是computed，什么watch，这两个有什么区别？

    1. computed：计算属性

      1. 计算属性会随原数据的变化而变化
      2. 计算属性是由data中的已知值，得到的一个新值。
      3. 计算属性不在data中，计算属性新值的相关已知值在data中。
      4. 别人变化影响我自己。

    2. .watch：监听数据的变化

      监听data中数据的变化
      监听的数据就是data中的已知值
      我的变化影响别人

    3. 区别 计算属性必须的有返回值，并且产生了一个新的值，且两个值互不影响，而侦听属性则没有必须的要求，侦听到数据变化可以做自己想做的操作，灵活性比较高，侦听器执行的方法可以接受两个参数，一个是未改变的值，一个是改变的值

    ​ 1.watch擅长处理的场景：一个数据影响多个数据

    ​ 2.computed擅长处理的场景：一个数据受多个数据影响
