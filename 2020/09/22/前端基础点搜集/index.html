js数据类型有哪些?有什么区别

数据类型可以分为基本数据类型和引用数据类型

基本数据类型 ：String、Number、Boolean 、Null、Undefined、Symbol、BigInt ;

引用数据类型：Object;

其中 Symbol、BigInt 是新增的数据类型.

基本类型—名值存储在栈内存中

引用数据类型–名存在栈内存中，值存在于堆内存中，但是栈内存会提供一个引用的地址指向堆内存中的值

不同类型的数据储存原理

栈：原始数据类型（Undefined，Null，Boolean，Number、String）

堆：引用数据类型（对象、数组和函数）

引用数据类型存储在堆(heap)中的对象,占据空间大、大小不固定,如果存储在栈中，将会影响程序运行的性能；引用数据类型在栈中存储了指针，该指针指向堆中该实体的起始地址。

当解释器寻找引用值时，会首先检索其在栈中的地址，取得地址后从堆中获得实体

为什么基本数据类型保存在栈中，而引用数据类型保存在堆中？

1）堆比栈大，栈比堆速度快；
2）基本数据类型比较稳定，而且相对来说占用的内存小；
3）引用数据类型大小是动态的，而且是无限的，引用值的大小会改变，不能把它放在栈中，否则会降低变量查找的速度，因此放在变量栈空间的值是该对象存储在堆中的地址，地址的大小是固定的，所以把它存储在栈中对变量性能无任何负面影响；
4）堆内存是无序存储，可以根据引用直接获取；

谈一下你对作用域的理解

作用域是指某个变量的使用(活动)范围

  • 全局作用域 : 定义的变量可以在代码(页面)的任何地方都能访问
  • 函数作用域(局部作用域) : 定义的变量只能在函数内部访问
  • 块级作用域 : ES6提供,通过 let, const 来体现,外层作用域无法获取到内层作用域,即使外层和内层都使用相同变量名，也都互不干扰

什么是作用域链

  • 由当所有作用域构成的一条链,在某个作用域下变量的有序访问.
    • 当访问某个变量时,会先在当前作用域查找是否有该变量,如果没有则向上一层作用域中查找,直到找到为止.

谈一下你对原型的理解

  • 原型是我们在定义函数时,javaScript自动创建的一个Object对象充当所定义函数的原型对象.
  • 每一个函数都有一个原型对象(显示原型),该原型对象上有一个constructor属性,指向该函数本身,通过该函数new出来的对象上面有一个隐式原型(_proto_)
    • 作用 :
      • 最重要的是共享属性和方法.
      • 节省内存空间
      • 实现继承
  • 原型链: 对象查找属性的过程,原型链的尽头是 Object.prototype—–null

什么是闭包

  • 闭包是由函数引用其周边状态（词法环境）绑在一起形成的（封装）组合结构.闭包可以让你从内部函数访问外部函数作用域。在 JavaScript 中，每当函数被创建，就会在函数生成时生成闭包。
  • 函数内部的数据,由于外部对其的引用,导致该函数在应用销毁的时候没有销毁,那么该函数则为闭包函数.
  • 使用: 只需要简单地将一个函数定义在另一个函数内部，并将它暴露出来。要暴露一个函数，可以将它返回或者传给其他函数。

如何修改函数的this指向,这些方法之间有什么区别?

this 的指向

  • 在浏览器里,在全局范围内this指向window对象;
  • 在函数中, this永远指向调用他的那个对象;
    • 普通函数中的this,也是指向全局对象(window),相当于window.函数名.
  • 在构造函数中,this指向new出来的那个新对象;
  • 对象中的this,永远指向调用该方法的对象;
  • 事件函数中的this,永远指向绑定事件的元素;
  • 在Node.js中,全局对象就是global;
  • call, apply, bind中的this被强绑定在指定的那个对象上;
  • 箭头函数中this比较特殊,箭头函数this为父作用域的this,不是调用时的this.

注: 要知道前四种方式,都是调用时确定,也就是动态的,而箭头函数的this指向是静态的,声明的时候就确定了下来.

修改函数的this执行

  • call , apply ,bind 这三个方法都可以用来更改this 的指向;
  • call 和apply 的区别在传参上,apll要的参数要以对象的形式进行传递,而call不需要;
  • call , apply 与 bind 的区别是 bind改变this 指向后,不会立即调用该函数,而是返回一个更改this后的新函数.而call ,aplly 在改变this指向的同时,就会调用该函数.
  • bind 返回的是一个新的函数，你必须调用它才会被执行。

事件委托或者事件代理的原理是什么?

​ 概念:事件委托是利用了事件流的冒泡机制来达到减少事件代码的一种编程技巧.
2. 原理:我们对一个HTML标签设置事件时,如果触发,因为本身页面有冒泡机制,那么父标签也会触发事件,那我们完全可以可以给父元素设置事件处理程序,在事件处理程序立面再判断点击的是哪个子标签,调用对应的代码.就子标签即不需要再定义事件处理程序了.
3. 使用
1. 给父元素(比如 body,document,或直接父标签)设置事件处理程序,
2. 在函数中借助event.target进行判断,判断用户点了哪一个标签,就执行对应的代码.
4. 好处
1. 减少代码

事件冒泡和事件捕获的区别是什么?

  1. 冒泡(推荐):先处理最具体的子标签,然后依次的向上处理父标签的事件,一直到window为
  2. 捕获:完全与冒泡相反,即先处理最顶层的标签的时间,即window,然后依次向下处理更具体的子标签的事件.

js异步编程方式有几种

  • ES6以前:
    • 回调函数
  • ES6: Promise
  • ES6: Generator
  • ES7: async await

promise有几种状态

  • 三种:
    • pending: 等待中(进行中),表示还未得到结果.
    • fulfilled: 已成功.表示已经得到正确的结果;
    • rejected: 已失败.表示已经得到结果,但并不是想要的结果.
