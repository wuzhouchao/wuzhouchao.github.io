react中什么是受控组件和不受控组件,以及他们有什么区别

不受控组件:

  • 指的是组件状态的改变不受 React 的控制。

  • 非受控组件具有以下几个特点：

    • 输入框没有 value 属性（单选框和复选框没有 checked 属性）；
    • 输入框内值的改变（单选框和复选框是否选中的改变）不会修改 state 或 props；
    • state 或 props 的改变不会影响输入框内的值（单选框和复选框的是否选中）；

受控组件:

  • 指的是组件状态的改变受 React 的控制,大部分情况下，react 都建议使用受控组件来实现表单效果。

  • 当我们为表单元素设置了 value/checked 属性时，就意味着表单元素已经变成了受控组件.

  • 特点与不受控相反

setState有什么特点

  • 不是立即更新。

  • 执行所有组件的didMount后，执行完父组件的didMount后，统一更新。

  • 更 新时会把多个更新合并，只更新一次。

  • 传入对象作为参数多次更新时，会忽略之前的只更新最后一次, 传入函数作为参数多次更新时，多次更新都会生效。

  • 不要直接修改 State而是应该使用 setState().

react列表渲染中的key有什么作用

key 帮助 React 识别哪些元素改变了，比如被添加或删除。

Key 应该具有稳定，可预测，以及列表内唯一的特质。不稳定的 key（比如通过 Math.random() 生成的）会导致许多组件实例和 DOM 节点被不必要地重新创建，这可能导致性能下降和子组件中的状态丢失

什么是react组件插槽(或者问你react如何实现组件复合)

组件可以在其输出中引用其他组件,

我们用同一组件来抽象出任意层次的细节。按钮，表单，对话框，甚至整个屏幕的内容：在 React 应用程序中，这些通常都会以组件的形式表示。

通常来说，每个新的 React 应用程序的顶层组件都是 App 组件。

什么是JSX,它的底层原理是什么?

js文件里的html标记就是JSX,通过React.createElement创建虚拟DOM

什么是虚拟DOM,为什么要使用虚拟DOM

在传统的 web 应用中，操作 DOM 一般都是直接更新操作的，页面节点每发生一次变化，都会导致整个页面的重绘和回流。

而 React 为了尽可能减少对 DOM 的操作，提供了一种和传统 DOM 操作完全不同的方式来更新 DOM*，从而代替了传统 *web 应用中直接的 DOM 操作，这种全新的操作 DOM 的方式就是–虚拟DOM。

vue中this.$set有什么作用

在我们使用vue进行开发的过程中，可能会遇到一种情况：当生成vue实例后，当再次给数据赋值时，有时候并不会自动更新到视图上去.也就是实际改变了数据但是没有触发视图更新.

调用this.$set()和Vue.set()方法(前者可以用在methods中使用),可以触发页面全部重新渲染。
